# EigenCapital Configuration Layout

This directory is the deployment-configuration source of truth for the
EigenCapital engine. The architecture splits the legacy monolithic YAML
(`paper_trading.yaml`) into domain-separated files under `configs/domains/`;
the legacy file is now a **derived mirror** regenerated by the toolchain.

**Rule of thumb**: edit domain files, not the mirror. See [Editing workflow](#editing-workflow) below.

## Composition order

The runtime loader (`PaperConfigRegistry.load()`) merges configuration
sources in the following order (last wins; the numbered layers map to
the concrete file groups in the table below):

```
         1. Domain promoted keys (domains/risk/*.yaml)
                  ↓
         2. Domain defaults overlay (domains/assets/_defaults.yaml)
                  ↓
         3. Per-asset file (domains/assets/<NAME>.yaml)
                  ↓
         4. Mode overlay (domains/modes/<mode>.yaml)
                  ↓
         5. Environment overlay (environments/<env>.yaml)
                  ↓
         6. Legacy paper_trading.yaml — unpromoted extras only
```

| Layer | Source | Scope |
|-------|--------|-------|
| **Risk** | `domains/risk/capital.yaml`, `sizing.yaml`, `exits.yaml`, `halt.yaml` | capital, drawdown limit, 28 sizing guardrails, adaptive-exit defaults, circuit-breaker thresholds |
| **Portfolio** | `domains/portfolio/weights.yaml` | weight_method selector, factor_exposure_limits |
| **ML** | `domains/ml/calibration.yaml`, `ensemble.yaml`, `meta_labeling.yaml`, `triple_barrier.yaml` | calibration method/bins, ensemble blend, meta-label thresholds, per-asset TP/SL for labels |
| **Broker** | `domains/broker/mt5.yaml` | MT5 bridge host/port, symbol map path |
| **Execution** | `domains/execution/spreads.yaml`, `sessions.yaml` | spread gate tier bps, session gate UTC hours |
| **Governance** | `domains/governance/regime_geometry.yaml`, `liquidity.yaml`, `narrative.yaml` | regime-conditional multipliers, liquidity regime thresholds, FXStreet narrative ingestion |
| **Infrastructure** | `domains/infrastructure/alerts.yaml` | PagerDuty/webhook alert channel configs |
| **Assets** | `domains/assets/_defaults.yaml` + `<NAME>.yaml` | per-asset allocation, pt_sl, adaptive_exit overrides |
| **Modes** | `domains/modes/<mode>.yaml` | mode-specific capital, sizing, drawdown, factor exposure limits |
| **Environments** | `environments/<env>.yaml` | data_source, rebalance, research_mode |
| **Legacy extras** | `configs/paper_trading.yaml` — unpromoted keys only | mode, modes, mt5, portfolio, optimizations, execution, rebalance, retrain_freq, retrain_window, research_mode, data_source |

### Upgrade precedence (promoted vs legacy_extras)

Keys are categorised at load time:

- **Promoted**: owned by a domain file. Changes to these keys must go
  through `configs/domains/`. If the mirror diverges on a promoted key,
  CI fails hard (exit 1 from `--check` / `--ci`).
- **Legacy_extras**: not yet migrated to a domain file. These are carried
  through from `paper_trading.yaml` as-is. Drift on legacy_extras is soft
  (CI warns but exits 0). See the [Phase 12.3 audit](#phase-12---hardening)
  for the list of pruned dead keys.
- **Pruned (Phase 12.3)**: `alerting`, `ensemble`, `calibration`, `kelly`,
  `meta_labeling` — were emitted by the registry but NEVER consumed
  through `EngineConfig` (no matching fields) or `defaults` (explicitly
  excluded). Their values in the YAML are dead; the registry no longer
  emits them.

## Editing workflow

### Quick-reference commands

| Task | Command |
|------|---------|
| **Validate the YAML tree** (schema + cross-fields) | `python tools/check_config_schema.py` |
| **Confirm mirror matches on-disk legacy** | `PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --check` |
| **Regenerate the legacy after a domain edit** | `PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --write` |
| **Inspect structural diff** | `PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --check` (drift in stderr) |
| **CI-mode diff** (structured JSON, categorised changes) | `PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --ci` |
| **Compare two configs** | `python tools/config_diff.py --from old.yaml --to new.yaml` |
| **Regenerate CONFIGURATION.md** (from domain models) | `python tools/config_docs.py` |
| **Schema-version bump** | Edit `configs/schema_version.json` (see [Schema versioning](#schema-versioning)) |

### Standard edit flow

1. **Identify** the domain file under `configs/domains/` that owns the
   setting you want to change. Use the tree above or the `summary()`
   output from the registry:
   ```python
   from configs.paper_config_registry import PaperConfigRegistry
   reg = PaperConfigRegistry.load()
   print(reg.summary())
   ```

2. **Make the edit**. Run `python tools/check_config_schema.py` to catch
   structural problems early.

3. **Regenerate the legacy mirror**:
   ```bash
   PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --write
   ```

4. **Run validation**:
   ```bash
   python tools/check_config_schema.py
   PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --check
   ```

5. **Commit both** the domain file change and the regenerated
   `configs/paper_trading.yaml`. CI fails if these diverge.

### Emergency override

If you need to hand-edit the legacy mirror (e.g., hot-fix in a deployed
environment where the toolchain is unavailable):

```bash
export ENABLE_LEGACY_EDITS=1
# Edit configs/paper_trading.yaml manually
# Then immediately run --write to re-sync:
PYTHONPATH=$PYTHONPATH:. python tools/config_mirror_legacy.py --write
unset ENABLE_LEGACY_EDITS
```

The `ENABLE_LEGACY_EDITS` env var suppresses the strict-write-mode
warning in `config_manager.py` and allows `--check` to pass without
the "operator edits should go to domains/" message. **Use sparingly.**

### Adding a new asset

1. Create `configs/domains/assets/<TICKER>.yaml` (ticker = MT5 symbol
   name, e.g. `AUDJPY.yaml`). Include the per-asset unique keys:
   ```yaml
   ticker: AUDJPY
   allocation: 0.05
   tp_mult: 2.01
   sl_mult: 0.52
   ```
2. Optionally override the shared defaults for this asset:
   ```yaml
   adaptive_exit:
     trail_activation_r: 0.6
     trail_retrace_pct: 0.50
   ```
3. Add the ticker to `configs/domains/assets/_index.yaml`.
4. Run the mirror + validation commands above.
5. Add the MT5 symbol mapping in `configs/mt5_symbol_map.yaml`.

### Removing an asset

1. Remove or comment out the per-asset file in `configs/domains/assets/`.
2. Remove the ticker from `configs/domains/assets/_index.yaml`.
3. Run the mirror + validation.
4. Move the model file to `paper_trading/models/orphaned/`.

## CI gate behaviour

### `config_mirror_legacy.py --check` (Phase 11.3)

- **Exit 0**: on-disk mirror matches the registry output (no drift).
- **Exit 1**: drift detected. Output includes:
  - A diff of the divergent paths
  - A notice directing the operator to edit domain files
  - The `ENABLE_LEGACY_EDITS` escape hatch

### `config_mirror_legacy.py --ci` (Phase 12.4)

Structured JSON output for CI consumption:

```json
{
  "drift": false,
  "summary": {"total": 0, "promoted": 0, "legacy_extras": 0, "other": 0},
  "changes": []
}
```

- **Promoted-key drift** (hard error — exit 1): a domain-owned key changed
  without a corresponding domain edit. The CI gate blocks the PR.
- **Legacy_extras drift** (soft — exit 0): an unpromoted key changed in the
  mirror. CI logs the change but does not block.

### Drift resolution

| Scenario | Root cause | Fix |
|----------|------------|-----|
| Promoted key changed | Operator edited the mirror directly | Revert the mirror edit, make the change in the domain file, run `--write` |
| Legacy_extras key changed | Intentional override in mirror | Accept the drift (soft gate), or promote the key to a domain file |
| Both drift | Domain file and mirror both changed | Resolve domain first, then re-run `--write` |
| CI fails on `--check` but mirror looks unedited | Permissions / encoding changed the file | Run `--write` from a clean checkout |

## Conventions

- **snake_case** for all keys and section names
- **Unit suffixes**: `_pct`, `_r`, `_bars`, `_secs`, `_bps`, `_usd`
- **Booleans**: use a leading `enabled:` section (not a `*_enabled` key at the
  top level)
- **Per-asset dynamics**: live in `domains/assets/<NAME>.yaml`, never inline
  in a profile
- **Domain files**: overwrite complete typed defaults; to restore a default,
  remove the line from the domain file (the typed default in the dataclass
  is the source of truth)
- **MT5 symbols**: map ticker → broker symbol in `configs/mt5_symbol_map.yaml`

## Schema versioning

`configs/schema_version.json` tracks the configuration schema version.
The current version is `2.0.0`. The version is checked at load time by
`PaperConfigRegistry` and by `check_config_schema.py`.

```json
{
  "version": "2.0.0",
  "compat_max": "<3",
  "previous_version": "1.0.0",
  "note": "Configuration architecture refactor complete. ..."
}
```

Bump the major version when backward-incompatible changes land (e.g.,
removing a promoted key). Bump the minor version for additive changes.
